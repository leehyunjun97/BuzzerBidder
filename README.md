# 🛎️ BuzzerBidder
**실시간 & 지연 경매를 지원하는 라이브 경매 플랫폼**

---

## 📌 프로젝트 개요

**BuzzerBidder**는  
실시간 입찰과 채팅을 중심으로 진행되는 **라이브 경매**와,  
기간을 두고 입찰하는 **지연 경매**를 모두 지원하는 웹 기반 경매 플랫폼입니다.

사용자는 실시간으로 입찰에 참여하고 경매 진행 상황을 즉시 확인할 수 있으며,  
경매 종료 전후까지 끊김 없는 사용자 경험을 제공하도록 설계되었습니다.

프론트엔드 팀장으로서  
- 전체 UI 구조 설계  
- 실시간 통신 구조 안정화  
- 사용자 실수 방지 UX 설계  

를 중심으로 프로젝트를 리딩했습니다.

## 🚀 주요 성과 및 문제 해결

### 1️⃣ 팀 리딩 & 개발 환경 구축

#### ▪ 백엔드 종속성 제거를 위한 선행 개발 구조
- 와이어프레임 기반으로 화면에 필요한 데이터 구조를 선제적으로 정의 
- API 미완성 상태에서도 개발이 멈추지 않도록  
  **프론트엔드 전용 Mock 데이터 환경 구축**

#### ▪ 개발 생산성을 위한 컨벤션 정립
- 폴더 구조 및 역할 분리 기준 수립  
- Prettier / ESLint 기반 코드 스타일 통일  
- 버튼, 입력창 등 공통 UI 컴포넌트 가이드 정리  
→ 팀원 간 코드 일관성과 유지보수성 향상  

---

### 2️⃣ 실시간 통신 안정화

#### ▪ 경매방 이동 시 연결 꼬임 문제 해결
- 현재 보고 있는 **경매방 ID**를 기준으로  
  - WebSocket 구독  
  - 구독 해제  
  - 재연결 흐름을 자동 관리  
- 잦은 경매방 이동 상황에서도  
  중복 연결 및 메시지 오류 방지  

#### ▪ 전역 실시간 알림 시스템 구현
- SSE 기반으로 페이지 이동 여부와 무관하게  
  중요한 이벤트를 즉시 사용자에게 전달  

---

### 3️⃣ 데이터 관리 & 성능 개선

#### ▪ 초기 로딩 속도 개선
- 화면 진입 후 API 호출 방식에서  
- 서버에서 데이터를 미리 주입하는 방식으로 전환  
→ 체감 로딩 속도 개선  

#### ▪ URL 기반 상태 동기화
- 검색 조건 / 필터 상태를 URL에 저장  
- 새로고침, 링크 공유 시에도 동일한 화면 유지  
→ UX 및 접근성 향상  

#### ▪ 복잡한 데이터 구조 통일
- 실시간 경매 / 지연 경매의 서로 다른 데이터 구조를  
- 화면에서는 동일한 형태로 사용할 수 있도록 변환 레이어 구현  
→ UI 로직 단순화 및 유지보수성 향상  

---

### 4️⃣ 사용자 실수 방지 & 안정성 강화

#### ▪ 입찰 방어 로직 구현
- 금액 오입력  
- 최소 입찰가 미달  
- 잔액 부족 상황  
→ 서버 요청 전 프론트엔드에서 선제 차단 및 안내  

#### ▪ SSR 환경 UI 오류 방지
- 이미지 캐러셀 등 화면 크기에 민감한 컴포넌트는  
  브라우저 환경에서만 렌더링하도록 분리  
- 서버 렌더링 시 발생하는 UI 깨짐 문제 해결
